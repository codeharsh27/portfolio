FitOne

A high-performance, local-first fitness tracking application.

Role

Lead Engineer

Timeline

2024

Architecture

Clean Arch + BLoC

Platform

Mobile (iOS/Android)

The Problem

Fitness enthusiasts need a reliable way to track their progress regardless of connectivity. Many modern apps rely heavily on cloud synchronization, causing latency and usability issues in gyms with poor signal.

The Solution

FitOne is engineered with a "Local-First" philosophy. By utilizing Hive for high-speed local storage, the app ensures zero-latency interactions. It implements Clean Architecture to strictly decouple business logic from UI code, ensuring scalability and testability.

Key Features

Offline-First

Seamlessly works without internet using Hive for local data persistence.

Clean Architecture

Scalable codebase separated into Domain, Data, and Presentation layers.

Advanced State

Predictable state management implementing the BLoC pattern.

Secure Auth

Firebase Authentication integrated for robust user management.

System Architecture

The app follows a strict Clean Architecture pattern to ensure separation of concerns. The Presentation Layer uses BLoC for predictable state management, communicating with the Domain Layer via UseCases. The Data Layer handles the offline-first logic, creating a Repository implementation that switches between Hive (Local) and Firestore (Remote) based on connectivity status, ensuring 100% uptime for the user.

Tools Used
